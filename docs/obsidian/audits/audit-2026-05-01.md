## 5. Recommendations (prioritised)

1. **Enable embeddings.** Run `npx gitnexus analyze --embeddings` to enable semantic search. This is the single biggest gap in index usability.
2. **Write CMS pipeline integration test.** The `__fixtures__/` directory has Strapi mock data. Write a test that validates `real_response.json → expected_dto` to lock the normalization contract.
3. **Add tests for home sections.** 8 home section components have no dedicated tests. Consider snapshot tests similar to `layouts.test.tsx`.
4. **Consider promoting `getCmsConfig` to typed contract.** It's the most-depended-on symbol in active code. A type alias would make signature drift visible at compile time.
5. **Document the Strapi bootstrap flow.** `seedContentManagerConfig`, `seedNavigationConfig`, `seedNavigationPermissions` run on every Strapi start. Their flow is indexed but not deeply documented.
6. **Keep `.gitnexusignore` up to date.** If new large generated/archived directories are added, exclude them.
